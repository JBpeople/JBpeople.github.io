Rust_Rust for Rustaceans：chapter1：内存区域

发表于2023-03-30|更新于2023-03-31|编程学习Rust for Rustaceans
|字数总计:2.8k|阅读时长:8分钟|阅读量:

最近在读《Rust for Rustaceans》,简单翻译一些里面讲解的概念。

并非所有的内存区域都是一样的，在大多数的编程语言环境中，你的程序可以访问栈、堆、寄存器、文本段、映射寄存器、映射文件，以及可能的非易失性 RAM。你选择使用哪个内存区域它将决定保持可访问状态的时间以及你用于访问它的机制产生影响。

内存术语

在深入了解内存区域之前先认识值，变量，指针的区别。

  • 值：在Rust中值是类型和类型值域中元素的组合。字符串”Hello world”是所有字符串域中的一个值，其表示形式为UTF-8编码。
  • 变量：存储值的最常见位置是变量，变量是堆栈上带有名称的值插槽。
  • 指针：指针是保存内存区域地址的值，因此指针指向一个位置。可以对指针进行解引用以访问存储在其指向的内存位置中的值。指针可以同时存在多份，指向同一个位置。

深入解析变量

前面的变量定义比较宽泛，遇到较为复杂的代码时需要更准确的模型来梳理程序运作方式。大体上就两种模型：高级模型和低级模型。高级模型在思考生命周期和借用层面的代码时非常有用，而低级模型则适用于当你推理不安全代码和原始指针时。

高级模型：不要把变量看作存储值的位置，将其看作是值的实例化，方便进行移动和使用。当你将一个值赋予给一个变量时，该值就被该变量命名。稍后访问变量就是在该变量之前的访问上画一条线到新的访问，从而建立两个访问的依赖关系，如果变量的值被移动就不能绘制任何线了。
在这个模型中，变量只存在于它持有合法值的时间内；你不能从一个值未初始化或已移动的变量绘制线条，因为实际上它不存在。使用这个模型，你的整个程序由许多这些依赖线组成，通常称为“流”，每个流追踪一个特定值的生命周期。当出现分支时，流可以分叉和合并，每个分支追踪该值的不同生命周期。

低级模型：变量命名内存位置，这些位置可能存储合法值，也可能不存储。你可以将变量视为“值槽”。指向变量的指针引用了变量的支持内存，并可以解引用以获取其值。在语句let x: usize中，变量x是一个在栈上具有usize大小值空间的内存区域的名称，尽管它没有一个明确定义的值（它的槽是空的）。如果你对该变量赋值，比如x = 6，那么该内存区域将保存表示值6的二进制位。&x指向x变量的内存地址，在你对x进行赋值时不会改变，但是x会重新寻找一块新的内存地址存储新的值，导致&x变成一个悬空指针，指向一块未知的内存。

建议两种模型都理解，这样可以从不同的角度去理解复杂代码。

内存区域

事实上电脑有很多内存区域，而编写Rust代码最重要的三个区域是栈、堆和静态内存。

栈：栈是一个内存段，在程序中用作函数调用的临时空间。每次调用函数时，在栈的顶部分配一个称为帧的连续内存块。在栈的底部附近是主函数的帧，当函数调用其他函数时，额外的帧被推入栈。函数的帧包含该函数中的所有变量以及函数所接受的任何参数。
当函数返回时，它的栈帧将被回收。函数本地变量值的字节不会立即被清除，但访问它们是不安全的，因为它们可能已经被后续函数调用覆盖，这些函数的帧与回收的帧重叠。即使它们没有被覆盖，它们也可能包含不合法使用的值，例如在函数返回时移动的值。
栈帧以及它们最终消失与 Rust 中生命周期的概念密切相关。在栈帧中存储的任何变量，在该帧消失后都无法访问，因此对它的任何引用都必须具有生命周期，该生命周期最多为该帧的生命周期。

堆：堆是一块与程序当前调用栈无关的内存池。堆中的值会一直存在，直到明确释放。当你希望某个值的生命周期超出当前函数框架的生命周期时，这是非常有用的。
堆允许你显式分配连续的内存段。当你这样做时，你会得到指向该内存段开头的指针。该内存段为你保留，直到你释放它。由于堆中的分配在函数返回时不会消失，因此你可以在一个地方为值分配内存，将指向它的指针传递给另一个线程，并使该线程安全地继续操作该值。换句话说，当你在堆上分配内存时，结果指针具有无约束的生命周期 ，它的生命周期取决于你的程序保持它处于活动状态的时间。
Rust 中与堆交互的主要机制是 Box 类型。当你编写Box::new(value)时，该值被放置在堆上，并且返回给你的（Box<T>）是该值在堆上的指针。当 Box 最终被丢弃时，该内存将被释放。
如果你忘记释放堆内存，它将永远存在，并且你的应用程序最终会耗尽计算机上的所有内存。这被称为内存泄漏，通常应该避免。但是，在某些情况下，你需要显式地泄漏内存。例如，假设你有一个只读配置，整个程序都应该能够访问它。你可以将其分配到堆上，并使用 Box::leak 显式地泄漏它以获得对其的'static引用。

静态内存：静态内存实际上是一个通用术语，用于描述编译程序时包含在文件中的几个紧密相关区域。当程序执行时，这些区域会自动加载到程序的内存中。静态内存中的值在整个程序执行期间都有效。程序的静态内存包含程序的二进制代码，通常被映射为只读。当程序执行时，它按指令逐条遍历文本段中的二进制代码，并在每次调用函数时跳转。静态内存还保存了你使用static关键字声明的变量的内存，以及代码中某些常量值，如字符串。
特殊生命周期'static由静态内存区域命名，标记引用为“只要静态内存存在”，即直到程序关闭。由于静态变量的内存在程序启动时分配，对静态内存中变量的引用被定义为 'static，因为它不会在程序关闭之前被释放。
在处理 Rust 时，你将比真正的静态内存（例如使用 static 关键字）更经常遇到'static生命周期。这是因为'static经常出现在类型参数的trait边界中。T: 'static 这样的边界表示类型参数 T 能够在我们保留它的时间内生存，直到程序执行结束。实质上，该约束要求 T 是自我拥有的，并且不借用其他（非静态）值，或者它所借用的任何值也是'static的，因此将一直存在直到程序结束。 'static 作为约束的一个很好的例子是 std::thread::spawn 函数，该函数创建一个新线程，需要你传递给它的闭包是 'static。由于新线程可能会超过当前线程的生命周期，因此新线程不能引用存储在旧线程堆栈上的任何值，而只能引用其整个生命周期都将存在的值，该生命周期可能等于程序的剩余持续时间。

[!note] 栈，堆和静态内存使用方法
栈：一般用于局部变量和函数调用。当你声明一个变量时，它会被分配到栈上，并在函数返回时自动释放。如果变量太大而无法放入栈中，例如数组或结构体，则需要将其放入堆中并使用指向堆内存的指针访问它们。

堆：一般用于动态分配内存，它可以存储大小不确定的数据，直到显式释放。如果你需要在程序运行时创建一个变量，但是不知道该变量的确切大小，则应该将其放入堆中。 示例：在堆中分配一个动态字符串。

静态内存：一般用于存储全局变量和常量，这些值在整个程序生命周期中都存在。Rust 中的静态内存通常包含程序的二进制代码、常量和静态变量。
